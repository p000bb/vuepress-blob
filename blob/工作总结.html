    前端工作总结

    熊猫宝宝 工作总结

    # 项目总结

    • # 松阳民情地图

    • 关键词： 云图初使用
    • 开发周期： 2个月(因最终没签署合同终止开发)
    • 开发心得： 进入公司第一个接手的项目，使用云图进行松阳民情地图的开发。初次使用云图，与之前的开发有所不同，刚开始有点不太适应，各种使用方法和api都不太熟悉。所以来说开发前期有点缓慢，需要花费一定的学习成本去学习云图的使用方法，后期因最终没签署合同终止开发。

    • # 数字三门

      • 关键词： 云图迭代升级、自定义场景开发、组件封装
      • 开发周期： 3~4月
      • 开发心得：
        1. 公司第一个正式使用云图开发的项目，开发过程中创哥一直在支撑，给我们提供解决方案和思路，不断地与创哥商讨需求，不停地优化和迭代云图，增强实用性
        2. 在开发过程中去设计和开发满足客户的需求，如实现自适应上下架，在后台设置指标的上下架状态，前端搭配自动显示和隐藏并且调整自己的位置。
        3. 云图优点：
        • 快速开发，熟练使用云图之后，开发某些大屏快、准、狠。依赖云图的特性，实现大屏开发，即开即用的特点，在客户时间紧急需要修改一些小的需求时，优点完美展示。
        • 组件封装，利用云图组件封装的特性，提前帮用户或开发封装号相应的组件，如部分公共图表，公共组件样式。再开发相应的页面时，托拉拽保存即可快速完成页面的开发。
        • 接手快，在修改他人编写的页面时，只需要修改需要修改的模块内部的代码即可，不需要了解整个页面的其他逻辑，十分敏捷。
        1. 云图不足点：
        • 需要一定的学习成本，再不熟练使用之前，开发效率缓慢
        • css样式污染，同一页面不同组件内部样式，相互冲突，没有进行组件之间的区分
        • 无法撤回，组件编写代码之后，关闭组件无法撤回之前的代码。
        • 没有历史记录，无法前端自己返回页面版本，需要联系后台进行数据库操作返回版本。
        • ……

    • # 智慧杭审

      • 关键词： 后台管理、首页大屏展示、独立开发二期
      • 开发周期： 6个月
      • 开发心得：
        1. 封装通用列表组件，继承搜索条件、列表、分页。传入相应的JSON配置页面实现快速开发
        2. 自我独立开发审计二期，并且维护和修改业主提供的建议
        3. 绩效考核算分模块，后台传给我相应的数据，前端自动生成层级表头，合并列表，从而满足业主的需求。
        4. 业主需求频繁变动，导致开发节奏混乱，不过最终还是静下心，加班加点完成业主的需求
        5. 总的来说审计项目还是比较顺利的，1期2期的开发，包括前期和业主面对面的需求探讨，以及后期与处室之前的需求探讨。去优化和迭代项目，参加想要的会议，有时会议展示自己修改的成果。通过这个项目，提升了自我的表达能力以及理解能力，理解相应的业务流程，从而保质保量的完成项目。

    • # 萧山区政务服务项目

      • 关键词： 云图开发、后台管理、大屏展示、app、独立开发
      • 开发周期： 4个月
      • 开发心得：
        1. 前期开发节奏紧凑，业主每天与我们探讨需求。每天按照业主需求进行相应的修改。在最初的一个月按时完成业主的前期需求，加班加点的完成大屏以及相应的后台管理系统，其中的交办流程，与需求后台反复商讨，也是在短时间内攻克开发。后期也开发了对应的app上架到浙政钉。
        2. 大屏展示数据下钻，市，区，部门，指数的展示。地图分区域显示场所码信息。
        3. 后台系统包括指标管理，预警管理，交办管理等模块，其中指标管理和大屏指标数据相互联动，预警管理，交办管理流程相互串通，查看相应的流程节点和详细信息。

    # 总结收获

    • 项目开发过程中，学习了新的技术，提升了自己的理解能力和业务能力
    • 使用云图开发，不断地去熟悉使用方法，并且找出它的不足之处，请求创哥这边支撑，不断优化云图，让它成为一个成熟的产品。
    • 多个项目独立开发，极大的提升了自己的项目搭建能力，以及开发能力。
    • 开发多个大屏项目，对大屏有着一定自我见解，开发效率敏捷，封装相应的图表组件，相互复用，快速开发。

    # 不足之处

    • 业务理解能力有待提升
    • 在开发过程中会存在和原型有部分出入的情况
    • 理解和表达能力欠缺
    • 开发考虑不够全面，开发过程中有一定的bug
    • 不善于使用外部资源，独自研究，耗费时间
